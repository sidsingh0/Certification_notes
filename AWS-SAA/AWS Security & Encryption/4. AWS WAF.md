Web Access Firewall
Works on Layer 7 (HTTP)
Not compatible with NLB (Layer 4)

Compatible with:
- ALB
- API Gateway
- Cloudfront
- Cognito User Pool

We can define a rule - WEB ACL
1. IP Address: Set upto 10000 Ips. Use more rules for more IPs
2. HTTP Headers, Body, Strings: Prevent XSS, SQL Injection
3. Size contraint: Server overloading
4. Geo Match: block countries
5. Rate based rules: DDoS protection

Web ACL are *regional*
Rule group is reusable set of rules that we can add to ACL

### Fixed IP while using WAF with Load Balancer
Sometimes we need fixed IP for some purposes
WAF works with ALB, but ALB Doesnt allow fixed IP
NLB has fixed IP but doesnt work with WAF

Solution:
Global Accelerator 
- gives 2 fixed anycast IPs
- routes user through closest region to AWS network
- sits in front of everything

# AWS Best Practices for DDoS Resiliency
- We should absorb attacks at edge and not at servers
	- Route 53 (First layer of defence)- 
		- Domain name resolution
		- Intelligent routing
		- **Can detect DDoS**
	- Cloudfront-
		- Caches content at edge -> Traffic is absorbed
		- Protects against UDP/SYN Flood
		- Most traffic never reaches our backend
	- Global Accelerator-
		- If our application does not have Cloudfront, we can use global accelerator
		- It has DDoS protection
		- Routes traffic through AWS network for lower latency
		- Routes traffic to nearest healthy region
	- API Gateway-
		- We can apply rate limiting/throttling
		- API Keys
		- Rquest validation
		- This prevents abuse of API
- To support these layers, we have
	- AWS WAF (Web application Firewall)
		- Blocks Bad Ips, bots and malicious requests
		- Blocks country (Pakistan), Unreputed IPs
- Valid traffic goes to our ALB, ASG and EC2 instances, they also absorb the attack since they can scale
- Keywords
	- “DDoS protection”
	- “protect at edge”
	- “absorb large traffic”

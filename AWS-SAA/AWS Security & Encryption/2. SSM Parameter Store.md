- It is a secure storage for secrets and configurations
- It is simpler and cheaper than AWS Secrets Manager
- Serverless, scalable, durable
- Optionally use KMS to encrypt. (SecureString data type)
- Version tracking of secrets and rollback
- IAM Security
- Notifications with EventBridge
- ![[Pasted image 20260424045145.png]]
- Standard tier is basically free for SSM parameter store
- Max size of parameter is 4kb and 8kb in advance
- Advance tier adds parameter policies
	- We can add TTL to the parameter (Should be deleted after x days)
	- Assign multiple policies at once

#### Parameter Store vs Secrets Manager

|                 | Parameter Store                  | Secrets Manager                                                                 |
| --------------- | -------------------------------- | ------------------------------------------------------------------------------- |
| Cost            | Very Cheap                       | Expensive (Per secret Cost)                                                     |
| Secret Rotation | Manual                           | Automatic with Lambda                                                           |
| Use case        | Simple Apps, <br>Flags, <br>URLs | DB Credentials, <br>API Keys that need to rotate, <br>security critical systems |
| Encryption      | KMS                              | KMS                                                                             |
- SecretsManager is mostly meant for integration with RDS
- AWS Secrets Manager – Multi-Region Secrets
	- Primary secret and Read Replica Secret(can be promoted to primary secret)
	- Use case: Multi region Apps/DBs, Disaster Recovery

- CloudWatch provides *metrics* for all the service.
	- CPUUtilization, Networking
- Metrics belong to namespaces.
- Dimensions are the metadata of a metric
	- InstanceId,Timestamp
	- We can define upto 30 dimensions for a metric
- We can also define custom metrics
	- Custom metrics are metrics that are not collected by cloudwatch by default
	- Ex.RamUtilization

#### Cloudwatch metric streams
- We can store metric over time as a stream
- This stream is stored using stream storing service like Kinesis Firehose.
- Firehose can take our stream of metric and store it into various target locations.
	- S3,
	- Opensearch(for indexing)
	- Redshift
- We can also filter the metrics while streaming it

# Cloudwatch Logs
- Log stream is a instance inside application which produces logs
- Log group is a name given to a group of logs(usually name of the application)
- Logs can expire(never expire - 1 day - 10 years)
- Amazon can send cloudwatch logs to the following services:
	- S3 (for storage)
	- Kinesis Data stream
	- Kinesis Fire hose(for storage)
	- Lambda
	- Opensearch(For indexing and searching)
- All logs are encrypted by default.
- We can setup KMS encryption too
- Sources:
	- Container logs inside ECS
	- Application logs (sent using SDK, Agent Logs)
	- Elastic Bean Stalk
	- Lambda
	- VPC flow logs
	- API Gateway
	- Cloudtrail
	- Route 53 (DNS Request logs)

#### Cloudwatch Logs for EC2
- By default, logs from EC2/On-premise dont automatically go to cloudwatch logs
- We need to provide write permission from EC2
- And write logs to an agent inside the EC2 (Logs Agent or Unified Agent)
- Types of agents:
	- Cloudwatch Logs agent
		- Old version, not used anymore
		- Can only send to Cloudwatch logs
	- Unified Log agent
		- Collects various other metrics like RAM usage, CPU Usage, processes etc.
		- Sends Logs+Metrics to Cloudwatch Logs
		- Configuration can be set for all our instances at one place (SSM)
		- Supported Metrics:
			- CPU (active, idle, guest, system user)
			- Disk Metrics (Available Storage), Reads, Writes, IOPS
			- RAM (free, inactive, used total, cached)
			- NetStat (Networking, number of UDP, TCP connections)
			- Processes (running, dead, blocked)
			- Swap Space (Free, used, used %) - It refers to disk stored as a backup RAM
		- There are also some out of the box metrics for EC2 - disk, CPU, network(high level)
#### Cloudwatch Insights
- We have stored our logs inside Cloudwatch Logs.
- Cloudwatch Insight allows us to query and search these logs.
- Features:
	- Search for specific things - Find logs with ERROR word
	- Filter Logs
	- Count and aggregate
	- Sort and limit results
	- Auto-parse logs (**It automatically understands the structure of your logs and turns them into fields you can query—without you writing parsing rules.**)
- Can query multiple log groups
- Uses SQL like query language but not SQL
- *It is not real time*

#### Exporting Cloudwatch Logs to S3
- API name for the task: `CreateExportTask`
- It can take upto 12 hours for log data to become available for export 
- This is not real time or near real time
	- For realtime logs, use Log subscription instead

# Cloudwatch Logs Subscription
- We can get real time log data from Cloudwatch Logs for analysis
- We can feed this to:
	- Kinesis Firehose (Store to S3)
	- Kinesis Data Streams
	- Lambda
- Subscription filter: Filter logs being sent to the destination

	- These logs are now sent to CloudWatch Logs Subscription Destination (Recipient account) in Account 2

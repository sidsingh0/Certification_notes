Features:
1. Schedule Cron Jobs
2. Receive events from amazon services
3. Trigger amazon services (Lambda SQS SNS)
Access Policy:
- We can manage access policy for all the buses in Event Bridge resource level policy
- We can allow events from all the accounts into one account for management
- Allow/deny events from different regions and accounts
#### Event Bridege Buses

Function
1. It recieves the events
2. Filters the events
3. Routes the events

Main feature:
1. Archive events
2. Replay archived events later

There are 3 buses in Amazon:
1. Default Bus
	1. Created with our amazon account
	2. Recieves all the events of our services
	3. Allows custom events
2. Custom Event Bus
	1. We create these busses for our applications
	2. Useful for access control (Since one application's bus doesnt need to see other application's bus)
	3. We can use these buses to organize events by domains
3. Partner Event Bus
	1. This event bus recieves events from third party applications(Shopify, SaaS applications etc.)
	2. External services can send events directly to our account

#### Schema registries
- Event Bridge Schema registries look at events
- The body of the event might contain details in json format like
	- {
		  "Amount": 100,
		  "Colour": "Yellow"
	  }
- Then schema registries automatically extract the schema for event types.
- We can then use these schemas while coding. Keeping teams aligned
- Features:
	- Versioning (Multiple versions of schemas can co-exist)
	- If event response changes, Schema Registry automatically captures new version and increaeses the version number

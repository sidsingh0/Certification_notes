- We can collect and aggregate logs from inside of container
- Can be used for:
	- EKS
	- ECS
	- Fargate
- ONLY FOR EKS and Kubernetes, an additional container containing the container insight agent is deployed to the pool. This agent is used for container discovery.

# CloudWatch Lambda Insights
- Collects metrics for Lambda functions
	- CPU usage
	- Ram usage
- Also tells us about cold starts and shutdowns

# Cloudwatch Contributor Insights
- Contributor refers to an user/ip/client
- Find “Top-N” Contributors through CloudWatch Logs
- VPC Flow logs is just network traffic logs
- There can be millions of lines inside of the VPC Flow Logs
- Now, Contributor Insights, looks into these logs and tells us important facts
- For example:
	- Problematic routes ex. /users has a lot of status 500, this can indicate an issue with the endpoint
	- Most appearing IPs, for example if an IP appears a lot of times, it can indicate attack like DDoS, or overloaded EC2 instance
	- Total Contributors(IPs) - Total number of clients
- We can create rules ourselves, to find a metric, or we can use sample rules from Contributor Insights

# CloudWatch Application Insights
- Generally when something breaks in our application, we go through the logs of EC2 and connected services like EBS, SQS, RDS etc. ourselves
- CloudWatch uses Sagemaker (ML) and does this for us
- It goes through logs of our application and connected services
- It also highlights what is working fine and what is not
- After detecting the issue, it can send it back to EventBridge to notify us

# Amazon Managed Streaming for Apache Kafka
- Apache kafka is an event streaming platform
- It stores large amounts of event data
	- Records are stored in disks called Topics
	- Topics are replicated in clusters
	- These clusters have multiple servers called *broker nodes*
	- These brokers work together to provide durability
	- To manage a cluster, Zoopkeeper is used
- Flow
	- Producer produces event (Kinesis, IOT, RDS, etc)
	- Events are stored in the topic 
	- and people subscribe to topic to see the stream (EMR, Kinesis, RDS, Sagemaker, S3)
- *It is an alternative to Amazon Kinesis*
- Managed Streaming for Apache Kafka is *Fully Managed*
	- Broker nodes are managed
	- Zookeeper nodes are managed
	- Auto recovery from failures
- Deploy the MSK cluster in your VPC, multi-AZ (up to 3 for HA)
- MSK serverless:
	- Run without managing the capacity- auto provision of resources

# Kinesis Data Stream vs MSK

|                      | Kinesis Data Stream         | Amazon MSK                         |
| -------------------- | --------------------------- | ---------------------------------- |
| Message size         | 1MB                         | 1~10MB                             |
| Data storage         | Data Streams with Shards    | Kafka Topics with Partitions       |
|                      | Shard Splitting and merging | Only partition addition is allowed |
| In flight encryption | TLS                         | Plaintext/TLS                      |
| At rest encryption   | KMS                         | KMS                                |
- **MSK** → “I want Kafka behavior, ecosystem, and control”
- **Kinesis** → “I want streaming with minimal ops”

# Amazon Managed Service for Apache Flink
- Apache flink is used to process streams
- For example "What is the avg temperature between now and 3:30 p.m."
- ![[Pasted image 20260421080133.png]]
- Flink cannot take input from firehose
- Flink can be used to transform data
- ❌ When NOT to use Flink
	- Simple ingestion → use Firehose
	- Simple trigger → use Lambda
	- Batch processing → use Glue
- Function:
	- Amazon MSK = event pipe (temporary storage)  
	- Amazon Managed Service for Apache Flink = processing brain 
	- Amazon S3 = long-term storage

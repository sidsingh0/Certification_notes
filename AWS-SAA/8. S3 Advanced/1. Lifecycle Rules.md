- Allows us to decide, what to do with files over time.
- We have 2 options:
	- Expiration Actions: 
		- configure objects to delete after some time. 
		- we can also delete older version of files (if versioning is enabled)
		- ex. delete logs older than 1 year.
	- Transition Actions: 
		- After specified periods, we can keep dropping the storage tier to save costs.
		- This can be done manually, or we can automate this using **lifecycle rules**.
- We can apply rules to a certain prefix (ex. s3://sid-bucket/mp4) or tags(ex. Finance Department)

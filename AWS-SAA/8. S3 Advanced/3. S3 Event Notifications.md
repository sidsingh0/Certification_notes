- For anything that happens,
	- S3:ObjectCreated
	- S3:ObjectRemoved
	- S3:Replication
	- and so on...
- We can catch these events and notify services to use these events.
- Use case: Creating thumbnails for all the photos uploaded to an S3 Bucket.
- These notifications can take some time (upto a minute) to be sent to the service
- Current Integrations:-
	1. Lambda
	2. SQS (Simple Queue Service)
	3. SNS (Simple Notification Service)
- NOTE: To enable S3 to access these services, instead of creating an IAM role for S3 and then allowing it to access Lambda, SQS, or SNS, 
  We define Lambda/SNS/SQS Access Policy for S3. These are just like bucket policies. and are written in json format.
- *Additional Integration: Eventbridge*
	- Enhances this notification capability
	- Captures all the events from S3
	- Allows even deeper filtering. Ex. by metadata, size, name etc. (using JSON)
	- Allows us to forward notifications to multiple (18) services
	- Allows us to replay, archive events etc.

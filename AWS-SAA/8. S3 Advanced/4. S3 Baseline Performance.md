### Optimizing performance
1. Multipart Upload
	1. Recommended for >100MB and MUST for >5GB
	2. Parallelizes uploads (Transfer speedup)
2. S3 Transfer Acceleration
	1. Allows us to upload file to edge locations (more than 200 across the world) using slow public network
	2. Then this edge location, transfers the file to target S3 region, using fast private network
	3. Compatible with Multipart Upload
3. S3 Byte range fetches (Opposite of multipart)
	1. Parallelize GETs by requesting specific bytes ranges (Read speed-up)
	2. Better resilience in case of failures 
	3. IMPORTANT USECASE: Get only partial file data (ex. first 50mb of the file)

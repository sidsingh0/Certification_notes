- Useful for network security
- They are like a firewall, allow in and out of our ec2 instances.
- By default,
	- Inbound rules for all ips are `blocked`
	- Outbound rules are `allowed` for all ips (to access internet)
- They are defined via
	- IPs and Ports
	- Other security groups
		- This assignment is generally used in load balancers.
		- Suppose we have,
		  FE VM (security group `F1`)
		  BE VM group (security group `BG1`)
		  We have allowed `F1`->`BG1`(Outbound) as well as `BG1`->`F1`(Inbound) 
		  Then, whenever a new BE instance is created by auto-scaling, we can just assign it to the `BG1` security group and it will be able to communicate with F1
- One security group can be applied on multiple instances
- One instance can have multiple security groups

- Security groups act as a wrapper, i.e. it is not like an application running on the VM,
  The VM does not know of the requests that are blocked
- If we get unlimited waiting/timeout issue -> security group issue
- If we get connection refused -> application issue
- They are locked down to region+vpc combination
- It is a good practice to maintain one separate security group just for SSH access
## Important Ports 
- 22 - SSH and 22 - SFTP
- 21 - FTP
- 80 - HTTP
- 443 - HTTPS
- 3389 - RDP

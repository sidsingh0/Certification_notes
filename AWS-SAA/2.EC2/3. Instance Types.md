Different instance types are optimized for different use cases.
### General Purpose
- They are balanced computes
- Useful for a lot of different types of tasks
- Ex. Code Servers, Repositories, Web Applications etc.
### Compute Optimized
- CPUs are optimized in this case.
- They are useful for CPU intensive tasks
- Ex. 
	- Scientific Modeling, Tough Mathematical Problems
	- High Performance Web Servers
	- Live Gaming Servers
	- Batch Processing
	- Video/Image Editing
	- Machine Learning/AI

### Storage Optimized
- Useful for high sequential read/write tasks
- Ex.
	- Transactional Systems
	- Large Databases (Relational & NoSQL)
	- Caching for in-memory databases (Redis etc.)
	- DFS(Distributed file systems)
	- Warehousing applications
### Memory Optimized
- Useful for fast in-memory tasks
- Ex.
	- Caching
	- In-memory Databases for BI
	- Applications performing real-time processing of big unstructured data

# Naming convention
- They are named like `m5.large`
	- `M` refers to the type of instance
	- `5` refers to the generation of compute(AWS continually improves instances)
	- `large` refers to the performance size of the particular instance (Higher the Better)

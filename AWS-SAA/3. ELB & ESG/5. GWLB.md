- Layer 3 / 4 (Transport and Network -> Packets)
- GWLB is used to manage and scale third party network services
- For example, Intrusion Detection System, Firewalls, Deep packet inspection, payload manipulation
- Generally, requests are like: User -> ALB -> App
- Using GWLB, we can first inspect the traffic and then allow it to contact out application,
  so all the network traffic goes through a service (which maybe hosted on EC2 as well) and then to our application.
- So the new flow is like: User -> routing tables -> GWLB -> Network Appliances ->
  back to GWLB(If accepted) -> Application

- It is a transparent gateway(since app doesn't know it exists): Single Entry/Exit
- Since it distributes the traffic, it is also a load balancer.
- Uses *GENEVE* protocol on 6081.

### Target Groups:-
1. EC2 instances
2. IP address (allows to use on premise device registered on the network)

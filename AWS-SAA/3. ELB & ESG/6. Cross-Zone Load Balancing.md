Suppose, 
`AZ A` has 8 Instances
`AZ B` has 2 Instances
Generally, load balancers forward requests from a zone, to instance in same zone.
(i.e. Request from `AZ A`, can not go to `AZ B`, by default)

This creates an imbalance in load.
`AZ A` instances handle:
50% ÷ 8 = **6.25% each** (underutilized)
Now `AZ B` instances must handle:
50% ÷ 2 = **25% each** (overutilized)

### Charges
- *ALB, by default, has Cross Zone Load balancing enabled. It has NO extra cost for enabling this feature*.
- *NLB, GWLB, have Cross Zone disabled. Enabling it, incurs charges.* 
- CLB, by default, has Cross Zone disabled, however it has NO extra charge for enabling.
- Also, Amazon charges for cross zone data transfer. ALB is not charged for cross-zone data transfer. CLB, NLB and GWLB are charged.

### When to disable
Possible reasons:
1. Reduce **cross-AZ traffic cost** (for NLB)
2. Strict **AZ isolation** (maybe compliance reason)
3. **Special networking architectures**
But for most web apps:
**Cross-zone is preferred.**

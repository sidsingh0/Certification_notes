- S3 keeps multiple copies of the same file over time. 
- Instead of overwriting a file, s3 stores history of changes.
- Use Cases:-
	- Accidental Deletion Protection
		- S3 does not delete the file immediately (By default)
		- Instead, It adds a **Delete Marker**
	- Easy rollback (We can go back to yesterday's version of the file)
		- Deleting a version = rollback mechanism
- Files that didnt have versioning before, and we enable versioning, their version is null
- If we **Suspended** versioning, existing versions remain, no new versions are created.
- Versioning is bucket level not file level.
- If we delete a version, It is permanent delete.
- In order to delete an object, we have to delete all the versions of the object.

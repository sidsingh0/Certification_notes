- Hybrid Cloud Storage (Using with on-premise storage)
- Application Hosting
- Media Hosting

### Buckets
- Can be seen as top level directories
- Files inside the bucket are called `objects`
- Buckets must have a globally unique name(across all the regions), although there is a new feature which allows for user-regional level.
- *However, S3 is defined in a region level. (not global)*

### Objects
- Objects are files and they have keys
- Objects are the values/body of the file.
	- Max object size is 5tb
	- Uploading files bigger than 5gb have to be uploaded as multi-part upload
	- Objects can have metadata (list of key value pairs)
	- Version Id (if version is enabled)
	- Tags (useful for lifecycle/security)
- Keys are the full path of the file.
- Key = prefix (folder1/folder2/) + object name (file.txt)
- There is no concept of directories, however the UI looks like one. Everything is just a key.

- We can host our static files (Html, css, images..) without servers/ec2 instances.
  This is called static website hosting.
- Mandatory files:
	1. error.html
	2. index.html
- Public URL: http://bucket-name.s3-website-region.amazonaws.com
- Static website works ONLY if:
	- Static hosting is enabled
	- AND bucket objects are publicly readable 
		- Bucket policy has `s3:getObjects` on principal: `*`
		- Block Public Access is turned OFF
- *If bucket is NOT publically readable, while static hosting is enabled, we get 403 Forbidden error*
- We have 2 types of endpoints:
	- API Endpoint: It serves the files inside the bucket (images, mp4, txt, etc..)
		- Example: https://bucket-name.s3.region.amazonaws.com/file.jpg
		- Default way s3 works.
		- Requires IAM permissions OR bucket policy
		- Used by EC2 accessing S3, Backend Applications, etc.
	- S3 Website Endpoint: It serves the static website
		- Example: https://bucket-name.s3.region.amazonaws.com/
		- It is only enabled when we turn on static hosting
		- Automatically serves index.html
		- Supports web-style routing (basic)
		- This endpoint does NOT use IAM authentication

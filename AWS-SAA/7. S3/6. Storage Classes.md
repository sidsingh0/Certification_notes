- Durability: How likely we are to lose our data.
  11 9s (i.e. 99.999999999% we wont lose our data) across all storage classes.
- Availability: How often we can access our data
  Depends on storage class

Classes:
1. S3 Standard (Hot Data)
	- Frequently accessed data
	- Low retrieval time
	- Low latency
	- High performance
	- High availability (99.999%)
	- *No retrieval cost!*
	- Use case:
		- Websites
		- Big data
		- Gaming
2. S3 Standard - Infrequent Access (Warm data)
	- Data is rarely accessed
	- But requires fast retrieval times
	- Retrieval cost applies
	- Cheaper than standard
	- 99.9% availability
	- Use case:
		- Backups
		- Disaster Recovery
3. S3 Standard - Single Zone IA (Cold Data)
	- Data not accessed frequently stored in a single AZ
	- If AZ fails, data is lost
	- 99.5% Availability
	- Cheapest IA option
	- Use case:
		- Secondary Backup
		- Recreatable data
4. Glacier Storage (Archive)
	- Very cheap data for long term backups
	- Retrieval Costs apply
	- Very slow Access
	- Types:
		1. Glacier Instant Retrieval
			- Millisecond Access
			- Minimum 90 day storage
			- Still cheap
		2. Glacier Flexible retrieval
			- Minutes to hours retrieval
			- Modes: Expedited, Standard, Bulk
			- Use case: You can wait for data
		3. Glacier Deep Archive
			- Slowest retrieval
			- Cheapest
			- Minimum 180 dats storage
			- Use case: Legal Archives, Compliance
5. S3 Intelligent Tiring
	- Small monitoring fee
	- No retrival cost
	- Automatic Optimization
	- AWS automatically moves data between tiers

Standard-IA → infrequent but regular use,business backups
Glacier Instant → almost never used, archive but fast

- *Storage classes are per object and not per bucket.*
- We can manually move the storage class per object

### S3 Lifecycle rules
- Automatic actions on S3 Objects over time
- There are 2 main actions for lifecycles:-
	1. Transition: Move objects to cheaper storage over time
	   Example:
		Day 0 → S3 Standard
		Day 30 → Standard-IA
		Day 90 → Glacier
		Day 180 → Deep Archive
	2. Expiration: Automatically delete objects after some time
	   Example: Delete logs after 365 days
- Lifecycle does not require versioning.

### Lifecycle vs Intelligent-Tiering

| Feature           | Lifecycle Rules  | Intelligent-Tiering |
| ----------------- | ---------------- | ------------------- |
| Control           | You define rules | AWS decides         |
| Complexity        | Manual setup     | Automatic           |
| Cost optimization | Yes              | Yes                 |
| Best for          | Predictable data | Unknown patterns    |

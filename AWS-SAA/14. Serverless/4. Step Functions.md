# Cognito
Cognito solves 2 problems:
1. **Who is this user?** - Logins - User Pool
2. **What AWS resources does this user have access to?** - Identity Pool

### User Pools (Authentication)
- Basically, it is like user table in our backend
- User pool manages user information + login system
- It handles:
	- Sign up
	- Sign in
	- Forgot password
	- Multi factor authentication
	- *SSO with Google/ Facebook/ Apple etc.*
	- Email/phone authentication
- Flow:
	1. User sends id password directly to User Pools, using our frontend
	2. User pool provides a JWT token to frontend
	3. Now, user calls our acual backend, using this JWT token
	4. Our backend -> *ALB / API Gateway integration*, verifies the token
	5. Allows based on result

### Identity Pools (Authorization)
- Sometimes, our user might want to access our AWS resources directly(without our backend), in that case we create an temporary identity for that user.
- *We create a shared IAM role for the user. (since a role can be assigned to multiple users at once)*
- This IAM role controls what user can access
- Here is the flow
	- User logs in using User Pools, Google, etc
	- Gets Token
	- Sends token to Identify Pool
	- Identity pool verifies the token with the issuer and gives iam role to the user using temporary credentials(AWS Security token service - STS)
	- User accesses the resource

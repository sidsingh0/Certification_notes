- API gateway acts as the door to backend
- Features:-
	- Request routing to
		- Lambda
		- HTTPS Endpoints - EC2, Container, ALB
		- AWS Service
			- start an AWS Step Function workflow, 
			- post a message to SQS
	- Authentication and Authorization
		- It can check JWT, IAM permissions etc.
		- Block users before they hit backend
	- Caching of common requests
	- Request validation
		- Required fields
		- Data formats
	- Request transformation
		- Convert the request to correct format (XML->HTTPS)
		- Rename fields
	- Support for Websockets and HTTP protocols
	- Rate limiting
	- Versioning of APIs (/v1/users)
	- *API Gateway + Lambda + DynamoDB= Serverless architecture*

#### Types of endpoints
1. Edge Optimized: 
	1. Global users
	2. Uses Cloudfront edge location for better latency
2. Regional:
	1. Regional users
	2. Could combine with cloudfront for caching etc.
3. Private
	1. Accesible only inside our VPC using an ENI
	2. Resource policy used to define access

#### API Gateway security
- User authentication:
	- IAM (Internal app)
	- Cognito (External users)
	- Custom Auth
- Connecting custom domain to API gateway(use ACM for https):-
	1. Edge Optimized(global): Create the certificate in us-east-1
	2. Regional endpoint: Create certificate in same region as the endpoint
	- Amazon assigns a default domain name to API gateway. (IP is not shown)
	  To assign domain name to  the resource, we need to use A-Alias(preferred) or CNAME record in Route 53

```
{
  "Version": "2012-10-17",
  "Id":"S3-Account-Permissions"
  "Statement": [
    {
      "Sid": "ReadAccessToBucket",
      "Effect": "Allow",
      "Principal": {
		"AWS": ["arn: aws: iam: : 123456789012 : root"]
	  },
      "Action": [
        "s3:GetObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::my-bucket",
        "arn:aws:s3:::my-bucket/*"
      ],
      "Condition": {
		  "IpAddress": {
		    "aws:SourceIp": "192.168.1.0/24"
		  }
	  }
    }
  ]
}
```
- Version: Policy language version
- Id: It is just a tag for the policy (optional)
- Statement: A set of rules
- Sid: Tag for the statement. (Useful for readability and auditing)
- Principal: Refers to which account should this statement be applied to
- Effect: Whether rule allows or denies the actions. (⚠️ **Explicit Deny always overrides Allow**)
- Actions: Specifies what operations are allowed or denied.
- Resource: Defines which AWS resources are affected.
- Condition: Add constraints. For example, deny access to S3 from home IP.(optional)
---
- (* star) is a wildcard in policies. For example:-
	-  "Action": [
			"iam:Get*",
			"iam:List*"
		]
	- This refers to all the get and list policies for IAM service.

## For protection of users
1. Implement User password policies (ex. allow user to reset password or not, update password after sometime, not use old passwords)
2. Enable MFA for all users(Password they know + Security device they own). This helps even if the password is compromised.
	- Device options for MFA-
		1. Virtual MFA device (ex. google auth, authy)
		2.  U2F(universal 2nd factor): It is like a pendrive to plug in(ex. Yubico)
		3.  Hardware Key FOB MFA device: The token is provided on the small display of the device (ex. Gemalto)

After creating a VM, we can connect to it using instance connect instead of SSH too.
This is built into the AWS portal, we just need to make sure SSH port is open. 
Instance connect automatically generates key value pairs and connects to the VM

### Important Note
Instance connect comes preinstalled with aws cli.
But to use this, we need to do `aws configure`, then enter our credentials
STOP!
This is a bad idea, since VM instance can be opened by multiple people, our credentials can get exposed.
Instead, use `IAM ROLES`, steps are as follows:-
- Go to IAM dashboard>Roles
- Create a role and give IAMReadOnlyAccess
- Go to EC2 dashboard
- Select the instance, then in actions > security > modify IAM role
- Add the created role
- DONE. we can list IAM users without using `aws configure`

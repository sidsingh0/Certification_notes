- It is a highly secure, powerful, rugged and portable device, designed *to collect and process data at the edge* and ultimately migrate data to AWS.
- When to use snowball:
	- Files are too big (Petabytes of data)
	- If uploading file takes more than a week (High network cost)
	- If there is no/bad internet connection
- Snowball is a offline data migration device. ![[Pasted image 20260417022644.png]]
- Client processes data on snowball -> ships it back to aws for migration to s3 bucket.
- *IMPORTANT: Snowball cannot directly migrate to Glacier. We will have to first upload the files to s3 bucket and then use lifecycle policies.*

# Edge Computing
- Dont confuse this with Edge locations (Points of presence)
- Edge computing refers to creating and processing data at locations where internet connection is not possible or bad
	- Ex. Moving truck (unstable internet)
	- Inside a mine (No internet)...
- Snowball has a mini cloud environment inside where you can setup EC2 and lambda functions to process your data.

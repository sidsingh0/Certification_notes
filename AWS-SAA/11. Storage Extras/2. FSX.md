- Fully Managed
- Shared file storage system (best for third party)
- EFS only works as simple Linux shared storage

- It can be accessed from
	1. EC2
	2. ECS
	3. EKS
	4. On-premise

- Use cases:
	- Lift and Shift Windows based applications
	- File settings and collaboration
	- High performance (HPC) workload -> Lustre
	- Backup and Disaster recovery 

*Supported loads:*
1. AWS NetAPP ONTAP
	1. SMB + NFS
	2. Enterprise migration from NetApp
	3. Netapp is CISCO's file management software
	4. Snapshots, replication, deduplication
	5. Exam trigger words:
		- “NetApp”
		- “Snapshots / replication”
		- “Hybrid cloud migration”
	
2. OpenZFS
	1. Linux workloads needing for NFS
	2. Want ZFS Features
	3. Exam trigger words
		- “ZFS”
		- “Low latency Linux workloads”
		- “NFS performance”
	
3. Windows File Server
	1. SMB, NTFS protocol needed
	2. Windows apps (IIS, .NET, AD-integrated application)
	3. Exam trigger words:
		- “Windows file share”
		- “Active Directory integration”
		- “SMB”
	
4. Lustre
	1. Used for distributed computing
	2. HPC/ML/Big Data
	3. *Need very high speed througput*
	4. *Native integration with S3*
	5. Exam trigger words:
		- “High performance”
		- “Machine learning training”
		- “HPC cluster”
		- “Millions of operations”

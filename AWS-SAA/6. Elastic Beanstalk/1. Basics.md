“I’ll give AWS my code… and AWS will handle all the infrastructure headache.”

Without Beanstalk: 
- You configure everything yourself  
-  You connect everything yourself  
-  You debug everything yourself
 This is painful and slow.

With Beanstalk, we just choose language and upload code. AWS automatically creates EC2 instances, Load Balancers, Auto Scaling and Monitoring.

### Components
1. Application: Container for everything
2. Version: Snapshot of our code
3. Environment: dev, test, prod

### Two types of environments (VERY EXAM IMPORTANT
1. Web Server Environment (normal apps)
	- User → Load Balancer → EC2
	- This is what 90% apps use
2. Worker Environment (background jobs)
	- App → Queue → Worker EC2 → Process task
	- Use case: Sending emails, Processing Images
	- No direct users → only processing tasks

### Deployment Modes
1. Single Instance
	- Useful for dev and test
	- No scaling
	- No Load Balancer
2. High Availability 
	- Load Balancer
	- Auto Scaling EC2 Group
	- Multi-AZ

- *Beanstalk is a free service, we only pay for underlying resources*
